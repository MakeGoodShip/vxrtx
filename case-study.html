Overview

What is vxrtx?

vxrtx is a Chrome side panel extension that uses AI to organize tabs and bookmarks. It detects stale tabs, finds duplicates, suggests intelligent groupings, and lets users lock folders and groups they want to protect from reorganization.

The extension supports five AI providers across three privacy tiers — from fully local inference with no network calls to full cloud context for maximum quality. A snapshot system provides safety nets with persistent history, import/export, and granular restore. The entire release pipeline is automated with CI gates, Release Please versioning, and Chrome Web Store draft uploads.

Settings

Privacy

Three Tiers, One Toggle

Privacy isn't a checkbox — it's a spectrum. vxrtx lets users choose exactly how much data leaves their machine, with clear labeling and instant switching between tiers.

Secure
Nothing leaves your machine

Use rule-based grouping, a local Ollama instance, or Chrome's built-in Gemini Nano. Zero network calls.

Relaxed
Cloud AI, minimal data

Send tab and bookmark titles to your chosen cloud AI — but never URLs. Enough context for smart grouping.

YOLO
Full context, maximum quality

Send titles, URLs, and timestamps. The AI sees everything for the best results. Your API key, your choice.

The tier system is enforced at the prompt engineering layer — each tier has its own prompt template that controls exactly what data is included. Switching tiers doesn't change the AI model, just the context window. API keys are stored locally in Chrome storage and never transmitted to vxrtx servers.

AI Providers

Use Whatever Works

vxrtx supports five AI providers spanning cloud, local, on-device, and no-AI options. Users bring their own API keys — there's no shared key, no middleman, no subscription.

OpenRouter Unified gateway to Claude, GPT-4o, Llama, Gemini, and 100+ models with one API key. Default provider for cloud tiers.
Direct API Bring your own Anthropic or OpenAI key for direct access without a middleman.
Ollama Run Llama 3.2, Mistral, Gemma 2, or Phi-3 locally. 100% private, no API key needed.
Chrome Built-in AI Gemini Nano on-device via Chrome 138+ Prompt API. Fully offline, no downloads, no subscription.
Rules Fast domain-based grouping with zero AI. Instant results, no network, no model — just pattern matching.

The provider architecture uses a shared interface with per-provider adapters. Smart model routing selects between fast and capable models based on item count. Timeout scaling adapts to collection size. All providers share the same prompt composition pipeline with modular system prompts and few-shot examples.

Design Polish

From Prototype to Product

The "Quiet Precision" design pass transformed the UI from a functional prototype into a cohesive product. No features changed — only the visual layer was elevated.

Design philosophy: The tool organizes chaos into order. The UI embodies that ethos — dense with information when needed, beautifully minimal when idle. Every surface intentional, every interaction grounded.

Key Design Decisions

Outfit Typeface

Geometric, confident, distinctive at every size. Replaces system defaults with a font that conveys precision.

SVG Icon System

All emoji replaced with crisp, consistent inline SVGs that scale perfectly and match the brand aesthetic.

Glowing Accents

CTA buttons and active nav indicators use box-shadow glow effects for a neon-on-dark signature.

Custom Form Controls

Range sliders with branded thumb glow, custom checkboxes, refined input focus states — all via CSS.

Semantic Color

Green for primary actions, indigo for relaxed tier, pink for YOLO, amber for undo/restore, red for destructive.

Noise Texture

Subtle SVG fractal noise at 1.8% opacity adds analog depth to the digital surface.

Patterns

Design System

A consistent set of patterns keeps the interface cohesive across all four screens and dozens of states.

State Machine Pattern

Both organizers follow an identical five-state workflow: idle → loading → preview → applying → done. This gives users a predictable mental model regardless of whether they're organizing tabs or bookmarks.

Progressive Disclosure

Complexity is revealed only when needed. Settings shows a minimal interface for the Secure tier, expanding to provider selection and API configuration only when users opt into cloud AI. The Bookmarks screen uses a mode menu rather than cramming all four tools into a single view.

Safety-First Operations

Every destructive action has a safety net. Organize operations create automatic snapshots. Delete requires confirmation. Applied changes can be undone. Folders and groups can be locked to exclude them from AI reorganization entirely.

Reflections

What Worked

Establishing a consistent state machine pattern early paid dividends — every new feature slots into the existing idle→loading→preview→done flow without inventing new paradigms. The tab and bookmark organizers are architecturally identical despite completely different Chrome APIs.

The three-phase brand evolution was unplanned but beneficial. Each color shift forced a re-evaluation of the entire interface, revealing places where the design relied too heavily on accent color versus structural contrast. The final palette — green primary, indigo and pink as semantic accents — gives each privacy tier its own visual identity.

Separating design polish from feature work prevented premature refinement from slowing development. The "Quiet Precision" pass happened only after all four screens were functional, making it possible to apply consistent treatment across the entire interface in one sweep.

Key takeaway: In a side panel extension, every pixel competes for attention in a 400px-wide strip. The most impactful design decisions are about what to remove — reducing visual noise, tightening spacing, and replacing emoji with purposeful iconography. Restraint is the hardest part of designing for constrained surfaces.
